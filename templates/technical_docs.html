Documentación Técnica para Desarrolladores

1. Arquitectura del Sistema

La aplicación utiliza el patrón MVT (Model-View-Template) basado en Flask. El backend gestiona la lógica financiera con una calificación de riesgo en escala A-D.

2. Modelado de Datos (SQLAlchemy)

Se utiliza SQLAlchemy como ORM (Object-Relational Mapper). Esto permite interactuar con la base de datos SQLite usando clases de Python en lugar de consultas SQL manuales.

  • Clase EmpresaDB: Representa la entidad principal. Almacena CUIT, Razón Social y flags de riesgo (Default/Quiebra).
  • Clase EECCDB: Representa los Estados Contables. Contiene una relación de clave foránea (ForeignKey) con la empresa. Aquí se guardan los saldos contables y los resultados del scoring.
  • Variable session: Es el objeto que gestiona la transacción con la base de datos (commit, rollback, add).

3. Lógica de Negocio y Scoring

El núcleo del análisis reside en dos funciones principales en app.py:

  • compute_metrics(balance): Toma un objeto de tipo balance y calcula ratios financieros brutos (ROE, Liquidez Corriente, EBITDA%, etc.). Retorna un diccionario estructurado.
  • calculate_scoring(metrics, ...): Aplica reglas de negocio. Transforma los ratios en puntajes del 0 al 10 basándose en umbrales definidos. Implementa los "Hard Filters" (como el rechazo por pérdida de PN > 80% o EBITDA Negativo Significativo vs PN).

4. Integración de IA con LangChain

Para la generación de descripciones automáticas, utilizamos el framework LangChain, que facilita la interacción con modelos de Google Gemini.

Componentes utilizados:
- ChatGoogleGenerativeAI: La interfaz que conecta con el modelo gemini-2.5-flash.
- ChatPromptTemplate: Define la "personalidad" del analista y estructura la instrucción (prompt) para asegurar respuestas consistentes.
- StrOutputParser: Limpia la respuesta del modelo para entregar texto plano al frontend.
- Chains (LCEL): Usamos el operador | para canalizar el prompt hacia el modelo y luego al parser (prompt | llm | parser).

5. Librerías Principales

Librería Uso en el Proyecto
Flask Servidor web, manejo de rutas (URL) y renderizado de plantillas.
SQLAlchemy Abstracción de la base de datos y gestión de relaciones 1-N.
LangChain Orquestación de la IA y gestión de Prompts.
python-dotenv Seguridad: Carga de claves de API desde el archivo .env.
CSV / IO Generación dinámica de reportes descargables para Machine Learning.

6. Variables de Entorno Críticas

El sistema requiere configuraciones externas para funcionar correctamente:

  • GOOGLE_API_KEY: Necesaria para que LangChain pueda autenticarse con los servicios de Google AI Studio.
